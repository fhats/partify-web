Admin¶

Contains functions and endpoints for administrative facilities.

partify.admin.admin_admin_update(*args, **kwargs)[source]¶

Updates the administrative privileges of other users given the input from the form presented by admin_console().

Route: /admin/admin_admin_update

Methods: POST

partify.admin.admin_console(*args, **kwargs)[source]¶

Shows the admin console. admin.html expects a list of forms for the admin_admin functions, a dict of user ids and names, a configuration form, the current user, and the privileges a user has.

Route: /admin

Methods: GET

partify.admin.admin_playback_pause(*args, **kwargs)[source]¶

Attempts to tell Mopidy to pause, regardless of Mopidy’s current play state.

Route: /admin/playback/pause

Methods: GET

partify.admin.admin_playback_play(*args, **kwargs)[source]¶

Attempts to tell Mopidy to play, regardless of Mopidy’s current play state.

Route: /admin/playback/play

Methods: GET

partify.admin.admin_playback_skip(*args, **kwargs)[source]¶

Skips the current track.

Route: /admin/playback/skip

Methods: GET

partify.admin.admin_queue_clear(*args, **kwargs)[source]¶

Clears the global play queue.

Route: /admin/queue/clear

Methods: GET

partify.admin.configuration_update(*args, **kwargs)[source]¶

Updates the configuration values provided by the configuration form presented by admin_console().

Route: /admin/config_update

Methods: POST

partify.admin.create_admin_admin_form(data=None)[source]¶

Returns a list of SingleUserAdminAdminForms - one for each user. This is useful for the Admin Admin section of the admin page, where one form per user is needed (due to the way WTForms does things).

partify.admin.make_admin_admin_form_object(user)[source]¶

Returns information about the user for use with an AdminAdminForm.
