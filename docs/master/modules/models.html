Models¶

Contains a bunch of SQLAlchemy Models for the data that Partify stores.

class partify.models.ConfigurationField(**kwargs)[source]¶

Represents a configuration field.

  • id: The ConfigurationField‘s unique ID
  • field_name: The name of the configuration field
  • field_value: The value the configuration field should have.
class partify.models.PlayHistoryEntry(**kwargs)[source]¶

Represents a log entry for a track that was played by a user. Used to generate statistics. These should stick around forever.

  • id: The PlayHistoryEntry‘s unique ID
  • track, track_id: The Track that was played
  • user, user_id: The User that played the track
  • time_played: The datetime that the Track started playing
class partify.models.PlayQueueEntry(**kwargs)[source]¶

Represents a playlist queue entry. These only live until the track they represent is played, then they are deleted.

  • id: The PlayQueueEntry‘s unique ID
  • track, track_id: The track that is represented by this PlayQueueEntry
  • user, user_id: The user that queued this track
  • mpd_id: The ID used by Mopidy internally for this queue entry
  • time_added: The datetime that this track was queued
  • user_priority: The priority of this track in the user’s queue
  • playback_priority: The playback priority in the global queue
class partify.models.Track(**kwargs)[source]¶

Represents track metadata. Used as a foreign key in PlayQueueEntry and PlayHistoryEntry. These should stick around forever and aid in faster metadata retrieval.

  • id: The Track‘s unique ID.
  • title: The title of the track.
  • artist: The artist that performs the track.
  • album: The album the track is found on.
  • length: The length of the track in seconds.
  • date: The date the track was released.
  • spotify_url: The URL of the track in Spotify’s catalog
class partify.models.User(name=None, username=None, password=None)[source]¶

Represents a User account.

  • id: The User‘s unique ID

  • name: The User‘s display name

  • username: The username of the user

  • password: The user’s password. Stored as a pass generated by

    Werkzeug’s generate_password_hash function, which computes an HMAC’d SHA1 hash of the input.

  • privs: A bitfield of the user’s privileges. See priv for definitions

class partify.models.Vote(**kwargs)[source]¶

Represents a vote that was taken by a user on the track. Once created these should not be removed, but instead simply updated.

  • id: The Vote‘s unique ID
  • user, user_id: The User that made this vote
  • pqe, pqe_id: The PlayQueueEntry (if any) which the user voted on
  • phe, phe_id: The PlayHistoryEntry (if any) which the user voted on
  • direction: The direction of the vote. Defaults to zero. -1 is down, 1 is up.
