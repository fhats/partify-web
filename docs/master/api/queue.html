Getting the Global Queue¶

Typically the global play queue is retrieved through the use of /player/status/poll by setting the parameter current to none. For more information, see Getting Player Status.

Adding Tracks to the Queue¶

Tracks are added to the user’s play queue using the /queue/add or /queue/add_album endpoints.

partify.queue.add_to_queue(*args, **kwargs)[source]

Takes a Spotify URL and adds it to the current play queue.

Parameters:spotify_uri (string) – The spotify URI of the track to add
Returns:The status of the request, the user’s queue, and the spotify url of the track added.
Return type:JSON string

Route: /queue/add

Methods: POST

partify.queue.add_album_from_track(*args, **kwargs)[source]

Takes a list of Spotify track URLs corresponding to an album and adds them to the user’s play queue.

Parameters:spotify_files (list of strings) – a list of spotify urls of the tracks in the album to add
Returns:The status of the request and the user’s queue.
Return type:JSON string

Route: /queue/add_album

Methods: POST

Reordering the User’s Queue¶

The user’s queue can be re-ordered by issuing a POST to /queue/reorder with a list of track:position mappings in the request body. The new positions will be updated in the database and will be updated in the Mopidy play queue on the next round of consistency checking.

Note

Right now it is preferable to provide every track in the user’s play queue and the new position for that track to avoid problems regarding tracks with the same priority in the user’s queue (this is what the built-in client currently does).

partify.queue.reorder_queue(*args, **kwargs)[source]

Reorders the user’s play queue.

Parameters:track_list (dictionary(integer, integer)) – A dictionary with keys of partify.models.PlayQueueEntry to be moved and values that are the positions to move the queue entry to.
Returns:The status of the request and the user’s queue
Return type:JSON String

Route: /queue/reorder

Methods: POST

Examples

POST /queue/reorder HTTP/1.1

44:1
50:12

52:3

58:9

Tracks can be removed from the user’s play queue by sending the queue entry’s ID in a POST request to /queue/remove.

partify.queue.remove_from_queue(*args, **kwargs)[source]

Removes the specified track from the user’s play queue.

Parameters:track_id (integer) – The ID of PlayQueueEntry to remove
Returns:The status of the request and the user’s queue
Return type:JSON string

Route: /queue/remove

Methods: POST
