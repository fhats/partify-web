Getting the User’s Vote Status¶

To find out what a User’s vote (if any) on a track is, use the /vote/status endpoint. The endpoint will return 1, -1, or 0 if the user has voted the track up, down, or has not voted yet (respectively).

partify.vote.vote_status(*args, **kwargs)[source]

Gets the logged-in user’s vote status for a certain track.

Parameters:pqe (integer) – The ID of the PlayQueueEntry
Returns:The status of the request and the direction of any existing vote.
Return type:JSON string

Route: /vote/status

Methods: GET

Note

A user cannot vote on a track he or she has queued; therefore, a user also cannot get his or her vote status on a track queued by him or her.

Example

GET /vote/status?pqe=5 HTTP/1.1
{
  "status": "ok",
  "direction": 1
}

Voting a Track Up¶

To vote a track up when a selection scheme that supports voting is enabled, send a POST to /vote/up with the ID of the PlayQueueEntry to vote on.

partify.vote.vote_up(*args, **kwargs)[source]

Records a vote in the up direction for the logged-in user.

Parameters:pqe (integer) – The ID of the PlayQueueEntry
Returns:The status of the request.
Return type:JSON string

Route: /vote/up

Methods: POST

Note

A user may not vote on a track her or she has queued.

Example

POST /vote/up HTTP/1.1

pqe: 5
{
  "status": "ok"
}

Getting the Vote Score for a Track¶

The total vote score of a PlayQueueEntry or PlayHistoryEntry may be retrieved by all users from the /vote/total endpoint.

partify.vote.vote_total()[source]

Retreives the total vote score for the specified track. One of pqe or phe must be specified, but not both.

Parameters:
  • pqe (integer, optional) – The ID of the PlayQueueEntry
  • phe (integer, optional) – The ID of the PlayHistoryEntry
Returns:

The status of the request and the total vote score of the specified track.
Return type:

JSON string

Route: /vote/total

Methods: GET

Examples

For a PlayQueueEntry

GET /vote/total?pqe=5 HTTP/1.1
{
  "status": "ok",
  "total": 6
}

For a PlayHistoryEntry

GET /vote/total?phe=5 HTTP/1.1
{
  "status": "ok",
  "total": -4
}
